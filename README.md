# DPI Dispatch

Mobile-first webapp to organize DPI's daily driver/vehicle dispatch — replacing the old
Airtable-requests → Google-Sheet → Viber workflow with one system.

Three roles, one app:
- **Requestors** (no login) — pick your name, submit a ride request, watch its live status.
- **Dispatcher** (passcode) — assign driver + vehicle + time, get warned on double-bookings,
  generate standing trips, share the day to drivers via Viber.
- **Drivers** (no login) — a personal link showing today's trips with map deep-links and
  Start / Done buttons.

Plus a searchable **history** with CSV export ("extract to Sheets").

## Stack

| Layer | Choice |
|---|---|
| Framework | Next.js 15 (App Router) + React 19 |
| Database | Postgres — local: Homebrew `postgresql@16`; prod: **Netlify DB** (Neon) |
| ORM | Prisma 6 |
| Deploy | Netlify (`@netlify/plugin-nextjs`) |

## Run locally

```bash
cp .env.example .env          # edit DATABASE_URL / ADMIN_PASSCODE if needed
npm install
npx prisma migrate dev        # create tables (first run)
npm run db:seed               # sample people, vehicles, one standing trip
npm run dev                   # http://localhost:3000
```

Local dev uses the Postgres already running via Homebrew. `npm run db:seed` prints the driver links.

## Environment variables

| Var | What |
|---|---|
| `DATABASE_URL` | Postgres connection string. Prod value is auto-set by `netlify db init`. |
| `ADMIN_PASSCODE` | Shared passcode that gates `/dispatch` and master-data edits. |
| `NEXT_PUBLIC_BASE_URL` | Base URL for driver links in Viber messages (driver links also use the live origin at runtime). |

## Routes

Persistent bottom navigation on every page: **Request · My trips · Schedule · More**
(+ a **Dispatch** tab that appears automatically on the dispatcher's device once the passcode
has been entered). The app is installable (PWA manifest + icons) — Add to Home Screen.

| Path | Who | What |
|---|---|---|
| `/` | Requestor | Pure request form (welcome card on first run; identity remembered; Today/Tomorrow chips; optional end time; live "free drivers" hint; extras collapsed under "Add details") |
| `/trips` | Requestor | My trips: live status list + plain-English status legend; success card after submitting |
| `/schedule` | Anyone | Read-only day **calendar** — time runs down, each driver a column, trips as blocks (where they'll be, who's free) |
| `/more` | Anyone | Hub: guide, history, driver finder (drivers tap their name once, then it's remembered), dispatcher links |
| `/guide` | Anyone | Onboarding: how to request, status colors, driver + dispatcher how-to, FAQ, Add to Home Screen |
| `/dispatch` | Dispatcher | Day board (List **or** Calendar view, choice remembered): incoming queue + assigned-by-driver, conflict detection, Viber share, generate standing trips |
| `/dispatch/manage` | Dispatcher | People, vehicles, standing-trip templates |
| `/driver/[token]` | Driver | Personal daily schedule + map links + Start/Done buttons |
| `/history` | Anyone | Filter/search past dispatches, export CSV |

## Deploy to Netlify

```bash
netlify init                  # link this folder to a new/existing Netlify site
netlify db init               # provision Netlify DB (Neon)
```

`netlify db init` sets `NETLIFY_DATABASE_URL` (pooled) and `NETLIFY_DATABASE_URL_UNPOOLED` (direct).
In the Netlify dashboard → Environment variables, add:

| Var | Value |
|---|---|
| `DATABASE_URL` | same as `NETLIFY_DATABASE_URL` (pooled — runtime) |
| `DIRECT_URL` | same as `NETLIFY_DATABASE_URL_UNPOOLED` (direct — migrations) |
| `ADMIN_PASSCODE` | your shared passcode |
| `NEXT_PUBLIC_BASE_URL` | the live site URL |

Then create the tables and (optionally) seed against production, and deploy:

```bash
npx prisma migrate deploy     # uses DIRECT_URL — creates tables in Netlify DB
netlify deploy --build --prod
```

The build runs `prisma generate && next build`.

## Data model (Prisma)

- **Person** — staff master; `isDriver` / `isAdmin` flags; `driverToken` for the driver link.
- **Vehicle** — vehicle master.
- **DispatchRequest** — the core record; carries the request *and* its assignment (driver,
  vehicle, scheduled time). A "trip" for a driver is derived (all requests for that driver + date).
- **RecurringTemplate** — standing trips, materialized per-date by the "Generate standing trips" button.
